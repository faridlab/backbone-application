# Glossary — Ubiquitous Language

> **Reader:** everyone. **Mode:** reference. One term, one definition, used consistently across the
> whole handbook. If a term you need is missing, add it here rather than coining a synonym.

## Framework & workspace

**Metaphor** — the meta-CLI and meta-workspace that manages a set of standalone project repos and
coordinates scaffolding, codegen, and runtime wiring across them. Each project keeps its own git
history. Binary: `metaphor` (`metaphor 0.2.0`).

**Backbone** — the framework itself: the suite of `backbone-*` crates plus the domain modules and
this service skeleton built on them.

**Workspace / consumer workspace** — a directory with a [`metaphor.yaml`](../../metaphor.yaml) that
registers projects. Product services live under `apps/<service>/`; upstream code is cloned
read-only under `modules/`. The standalone skeleton repo is **not** a workspace.

**`metaphor.yaml`** — the authoritative inventory of projects in a workspace (name, type, path).

**`metaphor.lock`** — resolved commit SHAs for pinned upstream refs. Regenerated by
`metaphor sync --update`; never hand-edited.

**Plugin** — a separate binary the CLI dispatches as a subprocess: `metaphor-codegen`,
`metaphor-schema`, `metaphor-dev`, `metaphor-agent`. See [ADR-0001](./adr/0001-subprocess-dispatched-plugins.md).

## Project types

**crate** — a focused Rust library (one crate, one concern), e.g. `backbone-core`.

**backbone-service** — a runnable HTTP/gRPC server that *composes* crates and modules. Has a
`main.rs` and a `[[bin]]`. **This repo is one.**

**module** — a domain library structured as a DDD 4-layer bounded context, where
`schema/models/*.model.yaml` is the source of truth. Owns business logic.

**cli-tool** — a subcommand dispatcher with plugin discovery (`metaphor-cli`).

**mobileapp** — a Kotlin Multiplatform + Compose client (`backbone-mobile`).

## Architecture

**Composition root** — the single place (here, [`src/main.rs`](../src/main.rs)) where independent
pieces are assembled into a runnable whole. This service is *only* a composition root.

**Layer / dependency rule** — Clean Architecture: dependencies point inward; `domain/` may not
import outer-layer crates (`sqlx`, `axum`, `tonic`, `reqwest`). See [architecture/layers.md](./architecture/layers.md).

**Bounded context** — a DDD boundary owning one coherent slice of the domain (Identity, File
storage, Accounting, …), each in its owning module. See [architecture/ddd.md](./architecture/ddd.md).

**Entity / value object / aggregate / repository** — DDD building blocks mapped 1:1 to folders under
`src/domain/`. Full definitions in [architecture/ddd.md](./architecture/ddd.md).

**Anti-corruption layer** — cross-context translation in `src/integration/context_map.rs`; keeps one
context's types from leaking into another.

## Codegen & regen safety

**Schema YAML / SSoT** — `schema/models/*.model.yaml` in a module; the single source of truth from
which entities, CRUD wiring, migrations, and DTOs are generated.

**`metaphor schema generate [--force]`** — regenerates `src/`, `migrations/`, `config/`, `tests/`
from the schema YAML. `--force` sweeps stale generated files.

**`user_owned:`** — the section in [`metaphor.codegen.yaml`](../metaphor.codegen.yaml) listing globs
the generator must never touch (read, merge, overwrite, or delete).

**`// <<< CUSTOM … // END CUSTOM`** — markers preserving a region *inside* a generator-emitted file
across regeneration. Must contain complete syntactic units.

**Generated vs hand-written folders** — parallel folders where the bare/singular name is generated
(`domain/entity/`, `application/service/`) and the plural/snake_case name is hand-written
(`domain/entities/`, `application/services/`). Definitive check: the `mod.rs` header.

## Runtime primitives

**`BackboneCrudHandler` / `GenericCrudService` / `GenericCrudRepository`** — the framework's
standard CRUD wiring (handler → service → repository). Use these before hand-rolling routes.

**`Module::builder().with_database(pool).build()?`** — the mandatory pattern for composing a domain
module into the service and obtaining its router.

**`MigrationManager`** — the in-process migration runner ([`src/infrastructure/database/migrations/`](../src/infrastructure/database/migrations/)),
wired in `main.rs`. Cross-module ordering is owned by `metaphor migration run-all`.

**Maintenance gate** — middleware (`backbone-maintenance`) that short-circuits traffic when the
service is in maintenance; exposes `/maintenance/status` (GET) and `/maintenance` (POST). Wired
outermost so it runs before any other layer pays its cost.

**Audit middleware** — `backbone-observability` layer wired innermost so it records the actual
response status the client sees.

**`AppConfig` / `validate_defaults`** — the config loader ([`src/configuration/`](../src/configuration/))
and the boot-time check that warns when a dev-default secret leaks into a non-dev environment.

## Subcommands (this binary)

**`serve`** — start the HTTP server (default). **`healthcheck`** — probe `/health`, exit 0 on 2xx
(used by Docker `HEALTHCHECK`). **`migrate`** — placeholder stub; real services delegate to
`metaphor migration run-all`.
